기본문법(strandard grammar): 태그(tag)

1. 글씨를 진하게 strong 사용

2. 밑줄 긋기 underline의 줄임말 u 사용

3. 제목 h1,h2,h3... h6 사용, 숫자가 커질수록 글씨는 작아짐.

4. br은 줄바꿈. br은 배운것들 중에서 특이하게 /br이 없음.

5. p는 단락(paragraph). br보다 훨씬 좋음.(br보다 가치있음)

6. br은 여러번 사용가능. p는 단락이 정해져있음.

7. style은 html의 속성.

8. tag와 tag사이의 여백 = margin이라 표현.(css에서의 용어)

9. margin옆에 있는 -top은 단락의 윗쪽에 여백을 만들겠다는 뜻.

10. px=pixel 줄임말. margin-top:숫자px은 숫자만큼 여백 생성.

11. 8~10번은 css용어를 사용하므로 아직은 몰라도됨.

13. tag의 이름만으로는 정보가 부족한게 있음.

14. 그래서 속성(attribute)을 사용. 2가지를 배움.(margin포함하면 3개)

15. source의 줄임말 src. 그리고 옆에 파일명을 적음.

16. width는 그림의 크기를 변경시킴.(width는 폭, 너비의 의미)

17. 이미지 사이트 unsplash(공공재 사이트)

18. li는 list의 줄임말로 목차.

19. li는 ul(unordered list,점이 생김,목차)이라는 부모(parent)tag를 가짐.

20. ul은 li라는 자식(child)tag를 가짐.

21. 극한(극단적 사고 필요)의 상황에서 코딩을 한다고 생각하자.

22. ol은 ordered list의 줄임말. 자동으로 목차의 숫자를 넘버링해줌.

문서의 구조와 tag의 슈퍼스타들

1. title은 웹페이지의 이름을 나타낸다.

2. meta는 웹 페이지의 보이지 않는 정보를 제공하는데 쓰이는 tag. (=저장된 방식)

3. meta의 속성인 charset은 문자규칙을 나타냄. 즉, utf-8로 나타내라 라는 것.

4. charest는 해당 문서를 어떤 방식으로 인코딩 할 수 있는지를 명시.(=문자 지원)

더 쉽게 말하면 동일 형식으로 문서를 읽어내라. 여기서 형식은 utf-8을 의미.

5. head는 본문을 설명하는 tag

6. body는 본문 tag

7. html은 head와 body를 감싸는 하나의 tag. 문서의 시작에 doctype html이라는 코드 필요.

html tag의 제왕

1. a는 anchor의 약자이고 link할 때 사용

2. a만 쓰면 link 안 걸림. href와 target, title 필요.

3. title의 경우 써도 되고 안 써도 되고. 사용 이유는 말 그대로 link의 제목

computer 2대 1. Web Browser, 2. Web Server

1. Web server는 http://info.cern.ch라는 주소를 가지고 있음. 그리고 index.html이 하드 디스크에 저장되어있음.

2. Web Browser는 http://info.cern.ch/index.html 즉 주소에서 index.html을 부른다.

3. 즉, Web Browser가 Web Server에게 request(요청), = client(의뢰인)

4. Web Server가 response(반응)

5. Web browser는 우리가 자주쓰는 naver, firefox, chrome 등 낯설지않음

6. Web Server는 추상적이고 낯섬.

7. 배우는 법: Web Server를 설치(어려움)하거나 대행자에게 맡기기(쉬움)

8. 즉, Web hosting으로 쉽게 배우고, Web Server로 원리를 파악.
